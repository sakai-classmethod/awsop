# 設計書

## 概要

awsop コマンドにおいて、`-h`オプションが動作しない問題を解決する。現在のコードでは`context_settings={"help_option_names": ["-h", "--help"]}`が設定されているが、実際には`-h`が機能していない。

調査の結果、Typer 0.20.0 では`context_settings`の設定方法が変更されている可能性がある。また、Typer は内部で Click を使用しており、Click の`help_option_names`設定が正しく伝播していない可能性がある。

## アーキテクチャ

### 現在の実装

```python
app = typer.Typer(
    add_completion=False,
    rich_markup_mode="rich",
    help="AWS credentials manager with 1Password integration",
    context_settings={"help_option_names": ["-h", "--help"]},
)
```

### 問題の原因

Typer では、`context_settings`を`typer.Typer()`に渡すだけでは不十分な場合がある。特に、コマンド関数に`@app.command()`デコレータを使用する場合、コマンドレベルでも`context_settings`を指定する必要がある可能性がある。

また、Typer 0.9 以降では、ヘルプオプションの処理方法が変更されており、`add_help_option`パラメータや`@app.callback()`の使用が推奨される場合がある。

## コンポーネントとインターフェース

### 修正対象

- `src/awsop/cli.py`: CLI アプリケーションの初期化とコマンド定義
- `src/awsop/shell/wrapper.py`: シェルラッパー関数の生成

### 修正方法の選択肢

#### 選択肢 1: コマンドレベルで context_settings を指定

```python
@app.command(context_settings={"help_option_names": ["-h", "--help"]})
def main(...):
    ...
```

#### 選択肢 2: callback を使用してアプリレベルで設定

```python
@app.callback(context_settings={"help_option_names": ["-h", "--help"]})
def callback():
    pass

@app.command()
def main(...):
    ...
```

#### 選択肢 3: 両方のレベルで設定

```python
app = typer.Typer(
    context_settings={"help_option_names": ["-h", "--help"]},
)

@app.command(context_settings={"help_option_names": ["-h", "--help"]})
def main(...):
    ...
```

## シェルラッパー関数の修正

### 現在の問題

シェルラッパー関数は、`command awsop "$@"`の出力を常に`eval`しようとする。ヘルプオプションが使用された場合、ヘルプメッセージが出力されるため、それを`eval`しようとして zsh のエラーが発生する。

### 修正方法

ヘルプオプション（`-h`、`--help`）、バージョンオプション（`-v`、`--version`）、リストオプション（`-l`、`--list-profiles`）などの情報表示オプションが使用された場合は、出力を`eval`せずにそのまま表示する。

```bash
function awsop() {
  # ヘルプ、バージョン、リストなどの情報表示オプションをチェック
  local info_option=false
  for arg in "$@"; do
    case "$arg" in
      -h|--help|-v|--version|-l|--list-profiles|--init-shell)
        info_option=true
        break
        ;;
    esac
  done

  if [[ "$info_option" == "true" ]]; then
    # 情報表示オプションの場合は eval せずに直接実行
    command awsop "$@"
    return $?
  fi

  # 通常の認証情報取得の場合は eval
  local output
  output=$(command awsop "$@")
  local exit_code=$?

  if [[ $exit_code -eq 0 ]]; then
    eval "$output"
  fi
  return $exit_code
}
```

## 実装の推奨アプローチ

### CLI の修正

調査の結果、**選択肢 2（callback を使用）**が最も確実な方法と考えられる。理由は以下の通り：

1. Typer のドキュメントでは、アプリレベルの設定には`@app.callback()`の使用が推奨されている
2. コマンドが 1 つしかない場合でも、callback を使用することで設定が確実に適用される
3. 将来的にコマンドを追加する場合も、一貫した動作が保証される

ただし、実装時には複数の方法を試して、最も効果的な方法を選択する。

### シェルラッパー関数の修正

情報表示オプション（`-h`、`--help`、`-v`、`--version`、`-l`、`--list-profiles`、`--init-shell`）が使用された場合は、出力を`eval`せずに直接実行する。
