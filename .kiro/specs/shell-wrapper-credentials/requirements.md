# 要件定義書

## はじめに

awsop コマンドにおいて、シェルラッパー関数経由で実行した場合の認証情報の設定に問題がある。現在の実装では、`--show-commands`（`-s`）オプションを指定した場合のみ export コマンドが標準出力に出力されるため、オプションなしで実行すると認証情報が環境変数に設定されない。

本仕様は、AWSume と同様の動作を実現することを目的とする。

### 現在の動作

- **`-s`なし**: export コマンドが出力されない → シェルラッパー関数が eval しても何も起こらない → 認証情報が設定されない ❌
- **`-s`あり**: export コマンドが標準出力に出力される → シェルラッパー関数が eval で実行 → 認証情報が設定される ✅（ただし画面には表示されない）

### 期待される動作（AWSume と同様）

- **デフォルト（`-s`なし）**: export コマンドを標準出力に出力 → シェルラッパー関数が eval して現在のシェルに設定 → 有効期限情報のみ画面に表示
- **`-s`あり**: export コマンドを標準エラー出力に表示（別シェルやスクリプトにコピペ可能） → シェルラッパー関数は画面表示のみで実行しない

### 背景

シェルラッパー関数は、Python CLI の標準出力をキャプチャして`eval`することで、現在のシェルセッションに環境変数を設定する。

- **デフォルト動作**: 現在のシェルに認証情報を設定する（最も一般的な使い方）
- **`--show-commands`オプション**: export コマンドを画面に表示して、別のシェルやスクリプトにコピペできるようにする（デバッグやスクリプト作成時に便利）

## 用語集

- **awsop**: AWS 認証情報を 1Password 経由で管理する CLI ツール
- **AWSume**: AWS 認証情報管理ツール。awsop の動作モデル
- **シェルラッパー関数**: Python CLI の出力を eval して環境変数を設定する zsh 関数
- **export コマンド**: シェル環境変数を設定するためのコマンド（例: `export AWS_ACCESS_KEY_ID=...`）
- **標準出力（stdout）**: プログラムの主要な出力先。シェルラッパー関数がキャプチャする
- **標準エラー出力（stderr）**: エラーメッセージや情報メッセージの出力先。画面に直接表示される

## 要件

### 要件 1: デフォルト動作（現在のシェルに認証情報を設定）

**ユーザーストーリー:** AWS 利用者として、シェルラッパー関数経由で awsop を実行した場合、オプションなしで現在のシェルに認証情報を設定したい。これにより、最も一般的な使い方がシンプルになる。

#### 受入基準

1. WHEN ユーザーがシェルラッパー関数経由で`awsop <profile>`を実行する THEN システムは export コマンドを標準出力に出力する
2. WHEN export コマンドが標準出力に出力される THEN シェルラッパー関数はそれを eval して環境変数を設定する
3. WHEN 環境変数が設定される THEN AWS CLI は設定された認証情報を使用できる
4. WHEN ユーザーが`-s`オプションなしで実行する THEN システムは有効期限情報のみを標準エラー出力に表示する
5. WHEN ユーザーが`-s`オプションなしで実行する THEN システムは export コマンドを画面に表示しない

### 要件 2: show-commands オプション（別シェルやスクリプトで使用）

**ユーザーストーリー:** AWS 利用者として、`--show-commands`オプションを指定した場合、export コマンドを画面で確認して別のシェルやスクリプトにコピペしたい。これにより、複数のシェルで同じ認証情報を使用できる。

#### 受入基準

1. WHEN ユーザーが`--show-commands`オプションを指定する THEN システムは export コマンドを標準エラー出力に表示する
2. WHEN ユーザーが`--show-commands`オプションを指定する THEN システムは有効期限情報も標準エラー出力に表示する
3. WHEN シェルラッパー関数経由で`--show-commands`を指定する THEN export コマンドが画面に表示される
4. WHEN シェルラッパー関数経由で`--show-commands`を指定する THEN シェルラッパー関数は標準エラー出力を無視する
5. WHEN ユーザーが画面に表示された export コマンドをコピーする THEN 別のシェルでペーストして実行できる

### 要件 3: シェルラッパー関数の更新

**ユーザーストーリー:** 開発者として、シェルラッパー関数が`--show-commands`オプションを特別扱いするようにしたい。これにより、export コマンドが画面に表示されるだけで eval されないようにする。

#### 受入基準

1. WHEN ユーザーが`--show-commands`オプションを指定する THEN シェルラッパー関数は情報表示オプションとして扱う
2. WHEN シェルラッパー関数が`--show-commands`を検出する THEN 標準出力をキャプチャせずに直接実行する
3. WHEN シェルラッパー関数が`--show-commands`を検出する THEN eval を実行しない
4. WHEN シェルラッパー関数が`--show-commands`なしで実行される THEN 標準出力をキャプチャして eval する

### 要件 4: 直接実行時の動作

**ユーザーストーリー:** AWS 利用者として、Python CLI を直接実行した場合（`command awsop`）、export コマンドを標準出力に取得したい。これにより、スクリプトから認証情報を取得できる。

#### 受入基準

1. WHEN ユーザーが`command awsop <profile>`を直接実行する THEN システムは export コマンドを標準出力に出力する
2. WHEN ユーザーが`command awsop <profile> -s`を直接実行する THEN システムは export コマンドを標準エラー出力に表示する
3. WHEN 直接実行する THEN システムは有効期限情報を標準エラー出力に表示する

### 要件 5: 情報表示オプションの動作

**ユーザーストーリー:** AWS 利用者として、`--help`、`--version`、`--list-profiles`などの情報表示オプションを使用した場合、export コマンドが出力されないようにしたい。これにより、シェルラッパー関数が誤って eval することを防げる。

#### 受入基準

1. WHEN ユーザーが`--help`オプションを指定する THEN システムは export コマンドを出力しない
2. WHEN ユーザーが`--version`オプションを指定する THEN システムは export コマンドを出力しない
3. WHEN ユーザーが`--list-profiles`オプションを指定する THEN システムは export コマンドを出力しない
4. WHEN ユーザーが`--unset`オプションを指定する THEN システムは unset コマンドを標準出力に出力する

### 要件 6: 後方互換性

**ユーザーストーリー:** 開発者として、既存のテストやスクリプトが壊れないようにしたい。これにより、安全に変更を適用できる。

#### 受入基準

1. WHEN 変更を適用する THEN 既存のプロパティベーステストを更新して通る
2. WHEN 変更を適用する THEN 既存の統合テストを更新して通る
3. WHEN `--show-commands`オプションを使用する THEN export コマンドが画面に表示される
4. WHEN シェルラッパー関数を使用する THEN 環境変数が正しく設定される
