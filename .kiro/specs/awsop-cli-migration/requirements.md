# 要件定義書

## はじめに

本ドキュメントは、既存の Zsh シェル関数 `awsop` を Python CLI ツールに移行するための要件を定義する。本システムは、1Password 連携による AWS 認証情報の取得と AssumeRole を行う。

### 背景

- 現在のシェル関数は Zsh に依存しており、保守性が低い
- Python CLI として実装することで、テストやパッケージ管理が容易になる
- AWSume は平文でアクセスキーを保存するため、社内セキュリティポリシーに違反し使用不可
- ジャンプアカウント運用で MFA 必須のため、1Password 連携は必須

### 目標

- 既存のシェル関数と同等の機能を提供
- セキュリティポリシー準拠: 認証情報は 1Password のみに保存
- モダンな UI（rich ライブラリ使用）
- uv でのインストール対応（pip は使用しない）

## 用語集

- **awsop**: AWS Operation の略。1Password 連携による AWS 認証情報管理ツール
- **1Password CLI**: 1Password のコマンドラインインターフェース（`op` コマンド）
- **AssumeRole**: AWS STS の機能で、一時的な認証情報を取得する API
- **一時認証情報**: AWS STS が発行する有効期限付きの認証情報（AccessKeyId、SecretAccessKey、SessionToken）
- **長期認証情報**: IAM ユーザーのアクセスキーや MFA 秘密鍵など、有効期限のない認証情報
- **プロファイル**: `~/.aws/config` に定義された AWS 設定の名前付きセット
- **シェルラッパー関数**: Python CLI の出力を eval して環境変数を設定するシェル関数

## 要件

### 要件 1: プロファイル切り替え

**ユーザーストーリー:** AWS 利用者として、プロファイル名を指定して AWS 認証情報を取得したい。これにより、複数の AWS アカウント間で素早く切り替えができる。

#### 受入基準

1. WHEN ユーザーがプロファイル名を指定して実行する THEN システムは `~/.aws/config` からそのプロファイルの設定を読み取る
2. WHEN プロファイルに role_arn が定義されている THEN システムはそのロールに対して AssumeRole を実行する
3. WHEN プロファイルに role_arn が定義されていない THEN システムはエラーメッセージを表示して終了する
4. WHEN AssumeRole が成功する THEN システムは一時認証情報を export コマンド形式で標準出力に出力する

### 要件 2: 認証情報の上書き

**ユーザーストーリー:** AWS 利用者として、異なるプロファイルに切り替える際に、前のプロファイルの認証情報が残らないようにしたい。これにより、意図しないアカウントでの操作を防げる。

#### 受入基準

1. WHEN ユーザーが新しいプロファイルを指定して実行する THEN システムは既存の AWS 関連環境変数を上書きする
2. WHEN 認証情報を出力する THEN システムは以下の環境変数を含める: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_SESSION_TOKEN, AWS_REGION, AWS_DEFAULT_REGION, AWS_PROFILE, AWSOP_PROFILE, AWSOP_EXPIRATION

### 要件 3: プロファイル一覧表示

**ユーザーストーリー:** AWS 利用者として、利用可能なプロファイルの一覧を確認したい。これにより、どのプロファイルが使用可能かを把握できる。

#### 受入基準

1. WHEN ユーザーが `--list-profiles` または `-l` オプションを指定する THEN システムは `~/.aws/config` からすべてのプロファイル名を抽出する
2. WHEN プロファイル名を抽出する THEN システムは各プロファイル名を 1 行ずつ標準出力に出力する

### 要件 4: export コマンド表示

**ユーザーストーリー:** AWS 利用者として、現在の認証情報を export コマンド形式で確認したい。これにより、手動で環境変数を設定する際の参考にできる。

#### 受入基準

1. WHEN ユーザーが `--show-commands` または `-s` オプションを指定する THEN システムは認証情報を export コマンド形式で標準出力に出力する
2. WHEN export コマンドを出力する THEN システムは以下の環境変数を含める: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_SESSION_TOKEN, AWS_REGION, AWS_DEFAULT_REGION, AWSOP_PROFILE, AWSOP_EXPIRATION

### 要件 4.1: 認証情報のクリア

**ユーザーストーリー:** AWS 利用者として、現在の認証情報をクリアしたい。これにより、セキュリティ上の理由で認証状態をリセットできる。

#### 受入基準

1. WHEN ユーザーが `--unset` または `-u` オプションを指定する THEN システムは環境変数を unset するコマンドを出力する
2. WHEN unset コマンドを出力する THEN システムは以下の環境変数を unset する: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_SESSION_TOKEN, AWS_REGION, AWS_DEFAULT_REGION, AWS_PROFILE, AWSOP_PROFILE, AWSOP_EXPIRATION

### 要件 4.2: リージョン指定

**ユーザーストーリー:** AWS 利用者として、プロファイルのデフォルトリージョンを上書きしたい。これにより、特定のリージョンで作業できる。

#### 受入基準

1. WHEN ユーザーが `--region` オプションでリージョンを指定する THEN システムは指定されたリージョンを AWS_REGION と AWS_DEFAULT_REGION に設定する
2. WHEN リージョンが指定されない THEN システムはプロファイルの region 設定を使用する
3. WHEN プロファイルに region が定義されていない THEN システムはデフォルトリージョン（ap-northeast-1）を使用する

### 要件 4.3: セッション名指定

**ユーザーストーリー:** AWS 利用者として、AssumeRole のセッション名をカスタマイズしたい。これにより、CloudTrail ログで操作を識別しやすくなる。

#### 受入基準

1. WHEN ユーザーが `--session-name` オプションでセッション名を指定する THEN システムは指定されたセッション名を使用する
2. WHEN セッション名が指定されない THEN システムは「awsop-<タイムスタンプ>」形式のセッション名を使用する

### 要件 4.4: ロール期間指定

**ユーザーストーリー:** AWS 利用者として、AssumeRole の有効期間をカスタマイズしたい。これにより、長時間の作業に対応できる。

#### 受入基準

1. WHEN ユーザーが `--role-duration` オプションで秒数を指定する THEN システムは指定された期間で AssumeRole を実行する
2. WHEN ロール期間が指定されない THEN システムはデフォルト期間（3600 秒）を使用する
3. WHEN 指定された期間が 43200 秒を超える THEN システムはエラーメッセージを表示して終了する

### 要件 4.5: MFA トークン指定

**ユーザーストーリー:** AWS 利用者として、MFA トークンをコマンドラインで指定したい。これにより、対話的なプロンプトをスキップできる。

#### 受入基準

1. WHEN ユーザーが `--mfa-token` オプションで MFA コードを指定する THEN システムは指定されたコードを使用する
2. WHEN MFA トークンが指定されない THEN システムは 1Password CLI 経由で MFA 認証を行う

### 要件 5: シェル統合

**ユーザーストーリー:** AWS 利用者として、シェルから直接 awsop を使用して環境変数を設定したい。これにより、eval を使った簡潔な構文で認証情報を適用できる。

#### 受入基準

1. WHEN ユーザーが `--init-shell` オプションを指定する THEN システムはシェルラッパー関数のコードを標準出力に出力する
2. WHEN シェルラッパー関数を出力する THEN システムは zsh 用の関数定義を含める
3. WHEN シェルラッパー関数を出力する THEN システムは zsh 補完スクリプトを含める
4. WHEN Python CLI が認証情報を出力する THEN システムは export コマンドを標準出力に出力し、ステータスメッセージを標準エラー出力に出力する

### 要件 4.6: 出力プロファイル

**ユーザーストーリー:** AWS 利用者として、取得した認証情報を `~/.aws/credentials` の特定のプロファイルに書き込みたい。これにより、他のツールから認証情報を利用できる。

#### 受入基準

1. WHEN ユーザーが `--output-profile` または `-o` オプションでプロファイル名を指定する THEN システムは取得した認証情報を `~/.aws/credentials` の指定されたプロファイルに書き込む
2. WHEN 出力プロファイルが既に存在し `manager = awsop` プロパティがない THEN システムはエラーメッセージを表示して終了する
3. WHEN 出力プロファイルが既に存在し `manager = awsop` プロパティがある THEN システムは認証情報を上書きする

### 要件 4.7: ロール ARN 指定

**ユーザーストーリー:** AWS 利用者として、プロファイルを使わずに直接ロール ARN を指定したい。これにより、一時的なロール切り替えが容易になる。

#### 受入基準

1. WHEN ユーザーが `--role-arn` オプションでロール ARN を指定する THEN システムは指定されたロールに対して AssumeRole を実行する
2. WHEN ロール ARN が指定される THEN システムは現在の認証情報をソースとして使用する
3. WHEN `--source-profile` オプションが指定される THEN システムは指定されたプロファイルの認証情報をソースとして使用する

### 要件 4.8: 外部 ID 指定

**ユーザーストーリー:** AWS 利用者として、AssumeRole に外部 ID を指定したい。これにより、サードパーティアカウントのロールを引き受けられる。

#### 受入基準

1. WHEN ユーザーが `--external-id` オプションで外部 ID を指定する THEN システムは AssumeRole リクエストに外部 ID を含める
2. WHEN 外部 ID が指定されない THEN システムは外部 ID なしで AssumeRole を実行する

### 要件 8: UI 表示

**ユーザーストーリー:** AWS 利用者として、認証情報取得の進行状況と結果を視覚的に確認したい。これにより、処理の状態を把握しやすくなる。

#### 受入基準

1. WHEN システムが 1Password 経由で認証情報を取得している THEN システムはスピナーアニメーションを標準エラー出力に表示する
2. WHEN 認証情報の取得が完了する THEN システムはプロファイル名と有効期限を色付きで標準エラー出力に表示する
3. WHEN エラーが発生する THEN システムはエラーメッセージを赤色で標準エラー出力に表示する

### 要件 6: 依存関係チェック

**ユーザーストーリー:** AWS 利用者として、必要なコマンドがインストールされているか確認したい。これにより、実行前に環境の問題を検出できる。

#### 受入基準

1. WHEN システムが起動する THEN システムは `op` コマンドの存在を確認する
2. WHEN 必要なコマンドが存在しない THEN システムはエラーメッセージを表示して終了する

### 要件 7: セキュリティ

**ユーザーストーリー:** セキュリティ管理者として、長期認証情報が平文で保存されないことを保証したい。これにより、セキュリティポリシーを遵守できる。

#### 受入基準

1. WHEN システムが認証情報を取得する THEN システムは 1Password CLI を使用して MFA 認証を行う
2. WHEN システムが認証情報を保存する THEN システムは長期認証情報を平文ファイルに保存しない
3. WHEN システムが一時認証情報を出力する THEN システムは標準出力にのみ出力し、ファイルには保存しない

### 要件 9: ログレベル

**ユーザーストーリー:** AWS 利用者として、デバッグ情報を表示したい。これにより、問題のトラブルシューティングができる。

#### 受入基準

1. WHEN ユーザーが `--info` オプションを指定する THEN システムは INFO レベルのログを標準エラー出力に表示する
2. WHEN ユーザーが `--debug` オプションを指定する THEN システムは DEBUG レベルのログを標準エラー出力に表示する
3. WHEN ログオプションが指定されない THEN システムは WARNING レベル以上のログのみを表示する

### 要件 10: 設定ファイル指定

**ユーザーストーリー:** AWS 利用者として、デフォルト以外の AWS 設定ファイルを使用したい。これにより、複数の設定セットを管理できる。

#### 受入基準

1. WHEN ユーザーが `--config-file` オプションでファイルパスを指定する THEN システムは指定されたファイルを AWS 設定ファイルとして使用する
2. WHEN ユーザーが `--credentials-file` オプションでファイルパスを指定する THEN システムは指定されたファイルを AWS 認証情報ファイルとして使用する
3. WHEN 設定ファイルオプションが指定されない THEN システムはデフォルトの `~/.aws/config` と `~/.aws/credentials` を使用する

### 要件 11: エラーハンドリング

**ユーザーストーリー:** AWS 利用者として、エラーが発生した際に原因を理解できるメッセージを受け取りたい。これにより、問題を自己解決できる。

#### 受入基準

1. WHEN 1Password CLI の実行が失敗する THEN システムは「1Password での認証に失敗しました」というメッセージを表示する
2. WHEN AssumeRole が失敗する THEN システムは「ロールの引き受けに失敗しました」というメッセージを表示する
3. WHEN AWS 設定ファイルの読み取りが失敗する THEN システムは「AWS 設定ファイルの読み取りに失敗しました」というメッセージを表示する
4. WHEN エラーが発生する THEN システムは終了コード 1 で終了する

### 要件 12: バージョン情報

**ユーザーストーリー:** AWS 利用者として、インストールされている awsop のバージョンを確認したい。これにより、最新版を使用しているか確認できる。

#### 受入基準

1. WHEN ユーザーが `--version` または `-v` オプションを指定する THEN システムはバージョン番号を標準出力に出力する
2. WHEN バージョン番号を出力する THEN システムは「awsop X.Y.Z」という形式で出力する

### 要件 13: ヘルプ表示

**ユーザーストーリー:** AWS 利用者として、awsop の使用方法を確認したい。これにより、利用可能なオプションを把握できる。

#### 受入基準

1. WHEN ユーザーが `--help` または `-h` オプションを指定する THEN システムは使用方法を標準出力に表示する
2. WHEN 使用方法を表示する THEN システムはすべてのオプションとその説明を含める
3. WHEN 引数なしで実行される THEN システムは使用方法を標準出力に表示する
