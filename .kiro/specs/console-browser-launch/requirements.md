# 要件定義書

## はじめに

本ドキュメントは、awsop に AWS マネジメントコンソールをブラウザで開く機能を追加するための要件を定義します。この機能により、ユーザーは指定したプロファイルの認証情報を使用して、ブラウザで直接 AWS コンソールにアクセスできるようになります。

## 用語集

- **awsop**: 1Password と連携して AWS 認証情報を管理する CLI ツール
- **コンソール URL**: AWS マネジメントコンソールにアクセスするための認証済み URL
- **サインイントークン**: AWS Federation Endpoint から取得する一時的な認証トークン
- **デスティネーション URL**: コンソールで開く最終的な AWS サービスの URL
- **Amazon Domain**: AWS リージョンに応じたドメイン（aws.amazon.com、amazonaws-us-gov.com、amazonaws.cn）
- **サービスマッピング**: サービスの短縮名から実際のコンソール URL へのマッピング

## 要件

### 要件 1: 基本的なコンソール起動

**ユーザーストーリー:** 開発者として、指定したプロファイルで AWS コンソールをブラウザで開きたい。これにより、CLI とコンソールの両方で同じ認証情報を使用できる。

#### 受入基準

1. WHEN ユーザーが`awsop <profile> -c`を実行する THEN システムは指定プロファイルの認証情報を取得し、AWS コンソールをデフォルトブラウザで開く
2. WHEN コンソールが開かれる THEN システムは AWS Federation Endpoint を使用してサインイントークンを取得する
3. WHEN サインイントークンを取得する THEN システムは一時認証情報（AccessKeyId、SecretAccessKey、SessionToken）を使用する
4. WHEN コンソール URL を生成する THEN システムはプロファイルのリージョン設定を反映する
5. WHEN ブラウザが正常に起動する THEN システムは成功メッセージを標準エラー出力に表示する

### 要件 2: サービス指定でのコンソール起動

**ユーザーストーリー:** 開発者として、特定の AWS サービスのコンソールページを直接開きたい。これにより、頻繁に使用するサービスに素早くアクセスできる。

#### 受入基準

1. WHEN ユーザーが`awsop <profile> -c <service>`を実行する THEN システムは指定されたサービスのコンソールページを開く
2. WHEN サービス名が短縮形で指定される THEN システムはサービスマッピングを使用して正しい URL に変換する
3. WHEN サービスマッピングに存在しないサービス名が指定される THEN システムはそのままサービス名を URL に使用する
4. WHEN サービス URL がテンプレート変数を含む THEN システムはリージョンとドメインを適切に置換する
5. WHEN 完全な URL が指定される THEN システムはその URL をデスティネーションとして使用する

### 要件 3: コンソール URL の出力

**ユーザーストーリー:** 開発者として、コンソール URL を取得してブラウザで開かずに他の用途に使用したい。これにより、URL を共有したりスクリプトで使用できる。

#### 受入基準

1. WHEN ユーザーが`awsop <profile> --console-link`を実行する THEN システムはコンソール URL を標準出力に出力し、ブラウザを開かない
2. WHEN ユーザーが`awsop <profile> --console-link <service>`を実行する THEN システムは指定サービスのコンソール URL を標準出力に出力する
3. WHEN URL が出力される THEN システムは他の情報メッセージを標準エラー出力に出力する
4. WHEN URL が出力される THEN システムは改行なしで出力し、パイプやリダイレクトで使いやすくする
5. WHEN URL が生成される THEN システムは完全な認証済み URL を生成する

### 要件 4: リージョン別のドメイン処理

**ユーザーストーリー:** 開発者として、GovCloud や中国リージョンでも正しいコンソール URL を生成したい。これにより、すべての AWS リージョンで機能を使用できる。

#### 受入基準

1. WHEN リージョンが`us-gov-`で始まる THEN システムは`amazonaws-us-gov.com`ドメインを使用する
2. WHEN リージョンが`cn-`で始まる THEN システムは`amazonaws.cn`ドメインを使用する
3. WHEN リージョンが標準リージョンである THEN システムは`aws.amazon.com`ドメインを使用する
4. WHEN ドメインが決定される THEN システムは Federation Endpoint とコンソール URL の両方で一貫したドメインを使用する
5. WHEN サービス URL テンプレートにドメイン変数が含まれる THEN システムは適切なドメインに置換する

### 要件 5: エラーハンドリング

**ユーザーストーリー:** 開発者として、コンソール起動が失敗した場合に明確なエラーメッセージを受け取りたい。これにより、問題を素早く特定して解決できる。

#### 受入基準

1. WHEN 認証情報の取得に失敗する THEN システムはエラーメッセージを表示し、終了コード 1 で終了する
2. WHEN サインイントークンの取得に失敗する THEN システムはエラーメッセージを表示し、終了コード 1 で終了する
3. WHEN ブラウザの起動に失敗する THEN システムはエラーメッセージと URL を表示し、ユーザーが手動でアクセスできるようにする
4. WHEN ネットワークエラーが発生する THEN システムは詳細なエラーメッセージを表示する
5. WHEN デバッグモードが有効である THEN システムはスタックトレースを含む詳細情報を表示する

### 要件 6: サービスマッピングのサポート

**ユーザーストーリー:** 開発者として、よく使うサービスの短縮名を使用してコンソールを開きたい。これにより、長いサービス名を入力する手間を省ける。

#### 受入基準

1. WHEN システムが初期化される THEN システムは一般的な AWS サービスの短縮名マッピングを持つ
2. WHEN 短縮名が使用される THEN システムは対応する完全なサービス名または URL パスに変換する
3. WHEN マッピングに含まれるサービスが指定される THEN システムは正しいコンソールページを開く
4. WHEN マッピングが完全な URL を含む THEN システムはその URL をテンプレートとして使用する
5. WHEN サービスマッピングが拡張される THEN システムは既存のマッピングを上書きせずに追加する

### 要件 7: 既存オプションとの統合

**ユーザーストーリー:** 開発者として、既存の awsop オプション（リージョン指定など）をコンソール起動と組み合わせて使用したい。これにより、柔軟な使い方ができる。

#### 受入基準

1. WHEN `--region`オプションが指定される THEN システムは指定されたリージョンのコンソールを開く
2. WHEN `--role-arn`オプションと`-c`が組み合わされる THEN システムは指定されたロールの認証情報でコンソールを開く
3. WHEN `--source-profile`オプションが使用される THEN システムはソースプロファイルの設定を考慮する
4. WHEN `--debug`オプションが有効である THEN システムは URL 生成プロセスの詳細ログを出力する
5. WHEN 複数のオプションが組み合わされる THEN システムは既存のオプション優先順位ルールに従う

### 要件 8: ユーザーインターフェース

**ユーザーストーリー:** 開発者として、コンソール起動の進行状況を視覚的に確認したい。これにより、処理が進行中であることを把握できる。

#### 受入基準

1. WHEN コンソール URL を生成中である THEN システムはスピナーと進行状況メッセージを表示する
2. WHEN ブラウザを起動する THEN システムは成功メッセージを表示する
3. WHEN URL を出力する THEN システムは情報メッセージを標準エラー出力に表示する
4. WHEN エラーが発生する THEN システムは Rich UI を使用してエラーメッセージを表示する
5. WHEN 処理が完了する THEN システムはスピナーを停止し、最終メッセージを表示する
