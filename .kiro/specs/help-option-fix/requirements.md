# 要件定義書

## はじめに

awsop コマンドにおいて、以下の 2 つの問題が発生している：

1. `--help`オプションは正常に動作するが、その短縮形である`-h`オプションが動作しない。ヘルプメッセージには`-h`が表示されているにもかかわらず、実際に使用すると「No such option: --h」というエラーが表示される。

2. シェルラッパー関数経由で`awsop --help`を実行すると、zsh のエラーメッセージが大量に表示され、ヘルプメッセージが正しく表示されない。これは、シェルラッパー関数がヘルプオプションを適切に処理していないことが原因である。

これらの不整合を解消し、ユーザーが期待通りにヘルプを表示できるようにする必要がある。

## 用語集

- **awsop**: AWS 認証情報を管理するコマンドラインツール
- **Typer**: Python のコマンドラインインターフェース構築ライブラリ
- **help_option_names**: Typer の設定パラメータで、ヘルプオプションの名前を指定する
- **context_settings**: Typer アプリケーションの動作を制御する設定辞書

## 要件

### 要件 1

**ユーザーストーリー:** ユーザーとして、`-h`オプションを使用してヘルプメッセージを表示したい。これにより、より少ないキー入力でヘルプ情報にアクセスできる。

#### 受入基準

1. WHEN ユーザーが`awsop -h`コマンドを実行する THEN システムはヘルプメッセージを表示する
2. WHEN ユーザーが`awsop --help`コマンドを実行する THEN システムは`-h`と同じヘルプメッセージを表示する
3. WHEN ヘルプメッセージが表示される THEN システムは`-h`を`--help`の短縮形として記載する
4. WHEN ユーザーが`awsop --h`コマンドを実行する THEN システムは適切なエラーメッセージを表示する

### 要件 2

**ユーザーストーリー:** 開発者として、Typer の設定が正しく適用されることを確認したい。これにより、将来的な設定変更が意図通りに動作することを保証できる。

#### 受入基準

1. WHEN Typer アプリケーションが初期化される THEN システムは`help_option_names`設定を正しく適用する
2. WHEN `context_settings`に`help_option_names`が含まれる THEN システムは指定されたすべてのヘルプオプション名を有効化する
3. WHEN ヘルプオプションが呼び出される THEN システムは他のコマンド処理をスキップしてヘルプを表示する

### 要件 3

**ユーザーストーリー:** ユーザーとして、一貫したコマンドライン体験を得たい。これにより、他の CLI ツールと同様の操作感で awsop を使用できる。

#### 受入基準

1. WHEN ユーザーが標準的な短縮オプション（`-h`、`-v`など）を使用する THEN システムは期待通りに動作する
2. WHEN ヘルプメッセージに短縮オプションが表示される THEN システムはその短縮オプションを実際にサポートする
3. WHEN ユーザーがサポートされていないオプションを使用する THEN システムは明確なエラーメッセージと代替案を提示する

### 要件 4

**ユーザーストーリー:** ユーザーとして、シェルラッパー関数経由でヘルプオプションを使用したい。これにより、どの環境でも一貫してヘルプを表示できる。

#### 受入基準

1. WHEN ユーザーがシェルラッパー関数経由で`awsop --help`を実行する THEN システムはヘルプメッセージを正しく表示する
2. WHEN ユーザーがシェルラッパー関数経由で`awsop -h`を実行する THEN システムはヘルプメッセージを正しく表示する
3. WHEN ヘルプオプションが使用される THEN シェルラッパー関数は出力を eval しない
4. WHEN ヘルプオプションが使用される THEN システムは zsh のエラーメッセージを表示しない
