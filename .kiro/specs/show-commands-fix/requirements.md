# 要件定義書

## はじめに

awsop の`-s`/`--show-commands`オプションが正常に動作していない問題を修正します。現在、このオプションは定義されているものの、実際の動作に影響を与えていません。本来の仕様（AWSume と同様）では、このオプションが指定された場合、有効期限情報を表示した後に、export コマンドを追加で出力する必要があります。

## 用語集

- **awsop**: AWS 認証情報を 1Password 経由で管理する CLI ツール
- **show-commands オプション**: `-s`または`--show-commands`フラグ。export コマンドを表示するためのオプション
- **export コマンド**: シェル環境変数を設定するためのコマンド（例: `export AWS_ACCESS_KEY_ID=...`）
- **有効期限情報**: 認証情報の有効期限を示すメッセージ（例: `[profile] Credentials will expire 2025-12-01 15:30:00 JST`）
- **ConsoleUI**: ユーザーへのメッセージ表示を担当する UI コンポーネント

## 要件

### 要件 1

**ユーザーストーリー:** CLI ユーザーとして、認証情報の export コマンドを明示的に確認したい

#### 受入基準

1. WHEN ユーザーが`-s`オプションを指定して awsop を実行する THEN システムは有効期限情報を標準エラー出力に出力する
2. WHEN ユーザーが`-s`オプションを指定して awsop を実行する THEN システムは有効期限情報の後に export コマンドを標準出力に出力する
3. WHEN ユーザーが`--show-commands`オプションを指定して awsop を実行する THEN システムは有効期限情報の後に export コマンドを標準出力に出力する
4. WHEN ユーザーが`-s`オプションなしで awsop を実行する THEN システムは有効期限情報のみを標準エラー出力に出力する
5. WHEN ユーザーが`-s`オプションなしで awsop を実行する THEN システムは export コマンドを出力しない

### 要件 2

**ユーザーストーリー:** 開発者として、show-commands オプションの動作を検証するテストを持ちたい

#### 受入基準

1. WHEN show-commands オプションが指定された場合 THEN テストは標準出力に export コマンドが含まれることを検証する
2. WHEN show-commands オプションが指定されない場合 THEN テストは標準出力に export コマンドが含まれないことを検証する
3. WHEN show-commands オプションが指定された場合 THEN テストは標準エラー出力に有効期限情報が含まれることを検証する
4. WHEN show-commands オプションが指定されない場合 THEN テストは標準エラー出力に有効期限情報が含まれることを検証する
