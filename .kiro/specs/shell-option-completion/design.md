# 設計ドキュメント

## 概要

本設計は、awsop コマンドの zsh シェル補完機能を拡張し、プロファイル名に加えてコマンドラインオプションも補完できるようにします。現在の`_awsop()`補完関数を拡張し、入力文脈に応じてオプションまたはプロファイルを適切に補完する文脈認識型の補完システムを実装します。

## アーキテクチャ

### システム構成

```
┌─────────────────────────────────────────┐
│  ユーザー入力（zshシェル）              │
│  例: awsop -<Tab>                       │
└──────────────┬──────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────┐
│  zsh補完システム                        │
│  - compdef による関数登録               │
│  - 補完トリガー検出                     │
└──────────────┬──────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────┐
│  _awsop() 補完関数                      │
│  ┌───────────────────────────────────┐  │
│  │ 1. 文脈解析                       │  │
│  │    - 現在の単語を取得             │  │
│  │    - 前の単語を確認               │  │
│  └───────────────────────────────────┘  │
│  ┌───────────────────────────────────┐  │
│  │ 2. 補完タイプ決定                 │  │
│  │    - オプション補完？             │  │
│  │    - プロファイル補完？           │  │
│  └───────────────────────────────────┘  │
│  ┌───────────────────────────────────┐  │
│  │ 3. 候補生成                       │  │
│  │    - オプションリスト             │  │
│  │    - プロファイルリスト           │  │
│  └───────────────────────────────────┘  │
│  ┌───────────────────────────────────┐  │
│  │ 4. 候補表示                       │  │
│  │    - _describe による表示         │  │
│  └───────────────────────────────────┘  │
└─────────────────────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────┐
│  ユーザーへの補完候補表示               │
│  - オプション + 説明                    │
│  - プロファイル名                       │
└─────────────────────────────────────────┘
```

### データフロー

1. **入力検出**: ユーザーが Tab キーを押すと、zsh が`_awsop()`関数を呼び出す
2. **文脈解析**: 現在の単語と前の単語を解析して補完タイプを決定
3. **候補生成**:
   - オプション補完の場合: 定義済みオプションリストから候補を生成
   - プロファイル補完の場合: `~/.aws/config`から候補を生成
4. **フィルタリング**: 現在の入力に一致する候補のみを抽出
5. **表示**: `_describe`を使用して候補を表示

## コンポーネントとインターフェース

### 1. generate_shell_wrapper() 関数

**場所**: `src/awsop/shell/wrapper.py`

**責務**: zsh シェルラッパー関数と補完スクリプトを生成する

**インターフェース**:

```python
def generate_shell_wrapper() -> str:
    """
    zsh用のシェルラッパー関数と補完スクリプトを生成

    Returns:
        str: シェルスクリプト文字列（awsop関数と_awsop補完関数を含む）
    """
```

**変更点**:

- オプションリストの定義を追加
- 文脈認識ロジックを追加
- プロファイル補完とオプション補完の切り替えロジックを追加

### 2. \_awsop() zsh 補完関数

**場所**: `generate_shell_wrapper()`が生成するシェルスクリプト内

**責務**: zsh の補完システムと統合し、適切な補完候補を提供する

**インターフェース**:

```zsh
_awsop() {
  # zsh補完システムから呼び出される
  # グローバル変数:
  #   - $words: コマンドライン上の単語の配列
  #   - $CURRENT: 現在補完中の単語のインデックス
  #   - $words[CURRENT]: 現在補完中の単語
  #   - $words[CURRENT-1]: 前の単語
}
```

**処理フロー**:

1. 現在の単語を取得（`$words[CURRENT]`）
2. 前の単語を取得（`$words[CURRENT-1]`）
3. 補完タイプを決定
4. 候補を生成してフィルタリング
5. `_describe`で表示

### 3. オプションリスト定義

**データ構造**:

## データモデル

### オプション定義

各オプションは以下の情報を持ちます：

```
オプション {
  短縮形: string (optional)  // 例: "-h"
  長形式: string             // 例: "--help"
  説明: string               // 例: "ヘルプを表示"
  値が必要: boolean          // 例: --region は true
}
```

### プロファイル情報

```
プロファイル {
  名前: string               // 例: "production"
  ソース: string             // 常に "~/.aws/config"
}
```

### 補完文脈

```
補完文脈 {
  現在の単語: string         // $words[CURRENT]
  前の単語: string           // $words[CURRENT-1]
  補完タイプ: enum {         // 決定された補完タイプ
    OPTION,                  // オプション補完
    PROFILE,                 // プロファイル補完
    PROFILE_FOR_OPTION       // オプションの値としてのプロファイル補完
  }
}
```

## 正確性プロパティ

_プロパティとは、システムのすべての有効な実行において真であるべき特性や動作のことです。これは人間が読める仕様と機械で検証可能な正確性保証の橋渡しとなります。_

### プロパティ 1: オプション補完の排他性と完全性

*任意の*補完文脈において、現在の単語が`-`で始まる場合、補完候補は CLI で定義されたすべてのオプション（短縮形と長形式の両方）を含み、かつプロファイル名を含まない
**検証: 要件 1.1, 1.2, 3.1, 4.1, 4.2**

### プロパティ 2: プロファイル補完の排他性

*任意の*補完文脈において、現在の単語が`-`で始まらず、かつ前の単語がプロファイル値を必要とするオプションでない場合、補完候補はプロファイル名のみを含み、オプションを含まない
**検証: 要件 2.1, 3.2**

### プロパティ 3: オプション説明の存在

*任意の*オプション補完候補において、各オプションには説明が含まれ、形式は`option:description`である
**検証: 要件 1.4, 4.3, 6.1, 6.2**

### プロパティ 4: プロファイル値補完

*任意の*補完文脈において、前の単語が`--source-profile`、`--output-profile`、`-p`、または`-o`の場合、補完候補はプロファイル名のみを含む
**検証: 要件 2.3, 3.4**

### プロパティ 5: 部分一致フィルタリング

*任意の*部分文字列入力に対して、補完候補はその部分文字列に一致するオプションまたはプロファイルのみを含む
**検証: 要件 1.3, 1.5**

### プロパティ 6: エラー耐性

*任意の*補完実行において、`~/.aws/config`ファイルが存在しない場合でも、補完関数はエラーを発生させずに空のプロファイルリストを返す
**検証: 要件 2.5**

## エラーハンドリング

### 1. AWS 設定ファイルが存在しない

**シナリオ**: `~/.aws/config`ファイルが存在しない

**処理**:

```zsh
if [[ -f ~/.aws/config ]]; then
  profiles=($(sed -nE 's/^\\[(profile )?([^]]+)\\]/\\2/p' ~/.aws/config))
else
  profiles=()  # 空の配列
fi
```

**結果**: プロファイル補完は空のリストを返すが、エラーは発生しない

### 2. 不正な文脈

**シナリオ**: 予期しない文脈で補完が呼び出される

**処理**: デフォルトでプロファイル補完を提供

**実装**:

```zsh
# デフォルトはプロファイル補完
local completion_type="profile"

# 文脈に応じて変更
if [[ $words[CURRENT] == -* ]]; then
  completion_type="option"
fi
```

### 3. 空の補完候補

**シナリオ**: フィルタリング後に候補が 0 件

**処理**: zsh の標準動作に従い、何も表示しない

**実装**: `_describe`が自動的に処理

### 文脈認識ロジック

```zsh
_awsop() {
  local -a profiles options
  local prev_word="${words[CURRENT-1]}"
  local current_word="${words[CURRENT]}"

  # オプションリストを定義
  options=(...)

  # プロファイルリストを取得
  if [[ -f ~/.aws/config ]]; then
    profiles=($(sed -nE 's/^\\[(profile )?([^]]+)\\]/\\2/p' ~/.aws/config))
  fi

  # 文脈に応じて補完タイプを決定
  if [[ "$prev_word" == "--source-profile" ]] || [[ "$prev_word" == "--output-profile" ]] || [[ "$prev_word" == "-p" ]] || [[ "$prev_word" == "-o" ]]; then
    # プロファイル名を必要とするオプションの後
    _describe 'profile' profiles
  elif [[ "$current_word" == -* ]]; then
    # 現在の単語が - で始まる場合はオプション補完
    _describe 'option' options
  else
    # それ以外はプロファイル補完
    _describe 'profile' profiles
  fi
}
```

### 後方互換性の保証

- 関数名は変更しない（`awsop()`, `_awsop()`）
- `awsop()`ラッパー関数のロジックは変更しない
- 既存のプロファイル補完動作を保持
- 新機能（オプション補完）は追加のみで、既存機能を削除しない

## パフォーマンス考慮事項

### プロファイル読み取りの最適化

- `~/.aws/config`の読み取りは補完のたびに実行される
- ファイルサイズが大きい場合、パフォーマンスに影響する可能性がある
- 現在の実装（`sed`コマンド）は十分高速であると想定

### オプションリストのメモリ使用

- オプションリストは静的に定義される
- 約 30 個のオプションで、メモリ使用量は無視できるレベル

## セキュリティ考慮事項

### ファイルパスの安全性

- `~/.aws/config`は固定パスを使用
- ユーザー入力からファイルパスを構築しない
- パストラバーサル攻撃のリスクなし

### コマンドインジェクション

- ユーザー入力は補完候補の選択のみに使用
- `eval`や動的なコマンド実行は行わない
- インジェクション攻撃のリスクなし

## 保守性

### オプションリストの更新

CLI にオプションを追加する際の手順：

1. `src/awsop/cli.py`にオプションを追加
2. `src/awsop/shell/wrapper.py`の`generate_shell_wrapper()`内のオプションリストに追加
3. テストを更新して新しいオプションを含める
4. ユーザーに`awsop --init-shell`の再実行を案内

### コードの可読性

- オプションリストは見やすい形式で定義
- 文脈認識ロジックはコメント付きで明確に記述
- zsh スクリプトは適切にインデント

## デプロイメント

### ユーザーへの影響

- 既存ユーザーは`awsop --init-shell`を再実行する必要がある
- 再実行しない場合、プロファイル補完のみが動作（後方互換性あり）
- 新機能を使用するには、シェル設定を更新する必要がある
